## MODULE 1: PUBLIC APPLICATION

### Scope

| Dimension | Definition |
|-----------|-----------|
| Company Scope | Per-company public pages; each PRV company has its own public-facing domain and branding |
| Data Isolation | Anonymous users see only company-approved public content; no cross-company data visible |
| Client Portal Scope | Authenticated client sees only their own projects, invoices, and documents within that company |
| Geographic Scope | No geographic restriction; accessible globally |
| Multi-Company | CEO-level admin can preview any company's public app; clients are always scoped to one company |
| Role Scope | Anonymous / Registered Public User / Authenticated Client — all scoped to one company context |

| Receipt / Proof of Purchase | System (on payment) | Document Center | 7 years |

## CROSS-MODULE SCOPE SUMMARY

Every module operates within the following universal scope framework:

```
PRV Group (root)
  └── Company [company_id]
        ├── Division / Department [division_id / department_id]
        │     ├── Team [team_id]
        │     │     └── Employee [user_id]
        │     └── Region [region_id] (Shop only)
        │           └── Store [store_id]
        └── Project [project_id]
              └── Site [location_id]
```

**Universal scope rules enforced at ALL modules:**

| Rule | Description |
|------|-------------|
| Company Isolation | No data crosses company boundaries without explicit multi-company grant |
| Row-Level Security | Database RLS enforces company_id filter on every query |
| Role-Based Visibility | Application layer enforces additional role + scope filters |
| No Data Leakage | Aggregated numbers in dashboards never reveal individual records the user cannot access directly |
| Audit Every Access | Every data access logged with user_id + scope at time of access |
| Scope Validated at Execution | Permission and scope checked at every action, not just at login |

| Single Store | All binary files stored once in Document Center; never duplicated |
| Reference Only | Modules store document_id references, never file content |
| Retention Mandatory | Every document category has a mandatory retention policy |
| Version Controlled | Every update creates a new version; previous versions preserved |
| Security Classified | Every document must be assigned a security level at creation |
| Audit Logged | Every view, download, share, and delete is logged |
| Company Scoped | Documents isolated per company; cross-company sharing requires explicit grant |

## CROSS-MODULE AUTOMATIONS SUMMARY

Every module's automations follow these universal rules:

| Rule | Description |
|------|-------------|
| Human Override | Every automation can be paused or overridden by an authorized human |
| Audit Trail | Every automation execution logged: trigger, action, result, timestamp |
| Company Scoped | Automations operate within company boundaries only |
| No Silent Failure | Automation failure generates a notification to Sysadmin |
| Idempotent | Automations designed to be safely re-executed if triggered twice |
| Configurable Thresholds | All threshold-based automations use company-configurable values |
| AI Suggestions | AI Center can suggest new automations but cannot activate without human approval |
| Disable Capability | Any automation can be disabled per company without code changes |
