Phase 07

Realtime Layer

Live data flowing into the Dynamic Island, dashboard KPIs, notification badge, and activity feed — via Supabase Realtime with Redis pub/sub fallback.

4-Tier Realtime Stack

Graceful degradation — each tier falls back to the next

Tier 1 — Primary
Supabase Realtime
Postgres postgres_changes listener. Zero polling. Sub-100ms latency. Already used by the presence system.
Used when: WebSocket available
Tier 2 — Fallback
Server-Sent Events
Next.js Route Handler streaming text/event-stream. One-directional, works through proxies. No WebSocket needed.
Used when: WebSocket blocked
Tier 3 — Bridge
Redis Pub/Sub
Upstash Redis pub/sub bridges server-side events to SSE stream. Already implemented in @prv/cache/pubsub.
Used when: Server bridge needed
Tier 4 — Last resort
Polling
TanStack Query with adaptive interval (5s active / 30s idle). Automatic when Realtime and SSE both unavailable.
Used when: All else fails

Technical spec

🔌
Hooks
  • useLiveKpis(companyId, role) — subscribes to kpis channel, returns live revenue/workforce/projects
  • useNotificationCount(userId, companyId) — live unread badge for tab bar
  • useActivityFeed(companyId, role) — prepends new events to timeline
  • useLiveContext(role, companyId, userId) — replaces placeholder in DynamicIslandBar
  • useLiveShift(userId) — live shift countdown + task assignment stream
📡
SSE endpoint (Tier 2)
  • GET /api/realtime/stream — auth-gated SSE stream per session
  • Subscribes to Redis pub/sub channels for the session's role + companyId
  • Streams text/event-stream — one JSON event per line
  • Heartbeat every 25s to keep connection alive through proxies
  • Client-side useSSEStream() hook with auto-reconnect
🗄
DB triggers (Tier 1 source)
  • Supabase DB webhook on invoices — fires revenue_tick on INSERT/UPDATE paid
  • Supabase DB webhook on notifications — fires notification_created on INSERT
  • Supabase DB webhook on projects — fires project_update on status change
  • Supabase DB webhook on company_memberships — fires workforce_change
⚡
Dynamic Island wiring
  • Replace static useLiveContext stub in dynamic-island-bar.tsx
  • Executive: primary="€1.84M" ticking from kpis:{companyId}
  • Worker: primary="Shift · 6h 24m" counting down from shift:{userId}
  • Store Manager: primary="€12.4K" from store:{storeId}
  • Specialist: primary="4.2K/min" from pipeline events
  • Auto-expand to warning state on critical indicator
