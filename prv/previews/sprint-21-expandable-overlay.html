Every action in PRV is validated through a 4-gate chain: Authentication → Authorization → Scope Validation → Audit Log. Face ID re-authentication is required for sensitive operations such as payroll approval, user deletion, and financial reports.
PRV connects to Stripe for payments and subscriptions, Supabase for database, auth, and realtime, Resend for transactional emails, and Typesense for fast scoped search. All integration credentials are stored in environment variables with zero hardcoded secrets.
Roles are hierarchical and scoped. A Superadmin sees all companies, a Regional Manager sees assigned stores, a Cashier sees only their register. Custom roles allow fine-grained per-module permissions with an enterprise-grade permission catalog.
Every read, write, and delete is logged to an append-only audit table with SHA-256 chain hashing to prevent tampering. Logs include user ID, role, scope, timestamp, action, and before/after state diffs. Audit logs are retained for 7 years.
