Architecture

DB Queries

  • queryNotifications(userId, companyId, filter, cursor)
  • queryNotificationCount(userId, companyId) — by type
  • markNotificationsRead(ids, userId)
  • dismissNotifications(ids, userId)
  • executeNotificationAction(id, action, userId)

API Routes

  • GET /api/notifications — paginated list, filter by type
  • PATCH /api/notifications/[id] — read, dismiss
  • POST /api/notifications/[id]/action — approve/decline
  • POST /api/notifications/bulk — mark-all-read, dismiss-all
  • GET /api/notifications/count — unread counts by category

Pages & Components

  • /notifications page — server SSR initial batch
  • NotificationsClient — realtime hydration via Phase 07 hook
  • NotificationCard — type-aware, expandable, action buttons
  • NotificationFilters — pill tabs with live counts
  • BulkActionsBar — mark-all, dismiss-all
  • EmptyState — per-filter variant

Interactions

  • Swipe left → dismiss (spring physics)
  • Swipe right → mark read
  • Tap to expand → inline action buttons
  • Approve/Decline inline (no navigation)
  • Realtime prepend (Phase 07 useActivityFeed)
  • Optimistic updates — instant UI before server confirms
